[v0](https://v0.dev/) is Vercel's AI-powered tool for generating React components and UI. v0 supports MCP via SSE (Server-Sent Events) transport, allowing it to connect to your memories.sh server.

## Quick Setup

### 1. Start the SSE Server

```bash
memories serve --sse --port 3030
```

This starts an HTTP server that v0 can connect to:

```
[memories] Starting MCP server with SSE transport
[memories] Listening on http://127.0.0.1:3030
[memories] Connect v0 or other clients to: http://127.0.0.1:3030/mcp
```

### 2. Expose the Server (for remote access)

If v0 is running remotely, expose your local server:

**Option A: ngrok (quick testing)**
```bash
ngrok http 3030
```

**Option B: Cloudflare Tunnel**
```bash
cloudflared tunnel --url http://localhost:3030
```

### 3. Configure v0

In v0's MCP settings, add:

```json
{
  "mcpServers": {
    "memories": {
      "url": "http://localhost:3030/mcp"
    }
  }
}
```

Or with a tunnel URL:
```json
{
  "mcpServers": {
    "memories": {
      "url": "https://your-tunnel-url.ngrok.io/mcp"
    }
  }
}
```

## Server Options

```bash
memories serve --sse [options]
```

| Option | Description | Default |
|--------|-------------|---------|
| `-p, --port <port>` | HTTP port | 3030 |
| `--host <host>` | Host to bind | 127.0.0.1 |
| `--cors` | Enable CORS | false |

### Examples

```bash
# Local only (default)
memories serve --sse

# Allow external connections (use with tunnel)
memories serve --sse --host 0.0.0.0

# Custom port with CORS
memories serve --sse --port 8080 --cors
```

## Endpoints

| Endpoint | Description |
|----------|-------------|
| `GET/POST /mcp` | MCP Streamable HTTP endpoint |
| `GET /health` | Health check |

## Security

<Callout type="warn">
The SSE server exposes your memories over HTTP. For remote access:
- Use HTTPS via a reverse proxy
- Consider IP allowlisting
- Don't expose sensitive memories
</Callout>

### Localhost Only (Recommended)

For local v0 CLI or desktop access, bind to localhost only (default):

```bash
memories serve --sse
# Only accessible from your machine
```

### With Tunnel

When using ngrok or Cloudflare Tunnel, anyone with the URL can access your memories. Use for temporary testing only, or add authentication via a reverse proxy.

## Alternative: Context Export

If you don't want to run a server, export context manually:

```bash
# Export rules for v0
memories prompt --types rule > v0-context.md

# Search for specific context
memories search "react" --type rule
```

Then paste into v0's chat.

## Troubleshooting

### Connection refused

Ensure the server is running:
```bash
memories serve --sse --port 3030
```

Test the health endpoint:
```bash
curl http://localhost:3030/health
# Should return: {"status":"ok"}
```

### CORS errors

Enable CORS for cross-origin requests:
```bash
memories serve --sse --cors
```

### v0 can't find memories

1. Check your memories exist: `memories list`
2. Verify the MCP connection in v0's settings
3. Try the `/health` endpoint to confirm server is reachable
